# Yu Wenyuan

Yu Wenyuan is presented in [[1-yi-token-julebu-jibaole-ai-de-ranliao-bugoule-duitan-yu-wenyuan-aliyun-bailian-jishu-fuzeren-ltn5k9jd9e04i5mfdkdo-ycoslsm]] as the technical leader for [[AliyunBailian]]. He explains the post-agent AI infrastructure problem from a cloud and MaaS platform perspective: demand is growing quickly, but usable AI depends on reliability, safety, latency, GPU scheduling, model quality, and cost-effective capacity conversion.

## Source Position

- Argues that token volume alone is misleading because different model types impose different compute and intelligence loads.
- Treats [[MaaSInfrastructure]] as an end-to-end system for turning GPU capacity into usable tokens.
- Takes the strong position that enterprises generally should not self-build model-serving stacks when platforms can provide model choice, security mechanisms, and utilization efficiency.
- Warns that AI-assisted programming still requires low-level computer knowledge, clear specifications, code review, and human responsibility.
- Sees agents and generative AI as major drivers of future token growth.
