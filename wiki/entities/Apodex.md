# Apodex

Apodex is the AI company discussed in [[e242-zuikuai-bannian-ai-paotong-zi-jinhua-yu-chen-tianqiao-shouxi-kexuejia-liaoliao-guigu-moxing-bi-zheng-zhi-di]]. The source presents it as a [[ChenTianqiao]]-backed and strategically directed company focused on Heavy Duty Solver work rather than consumer chat, image generation, or video generation. Its target is a [[DiscoveryModel]]: a system that can propose hard scientific hypotheses, search and reason about them, write code or run simulations, and verify whether the hypotheses hold.

The episode describes [[DuShaolei]] as Apodex's chief scientist for training and inference, and [[LiBeibin]] as chief scientist for code and self-evolution. Their shared technical path connects [[DeepResearch]], agent-team problem solving, [[AIVerification]], [[RecursiveSelfImprovement]], and [[ResearchTaste]]. The company is also described as using a model post-trained from [[Qwen]] for Deep Research-style capabilities.

## Source Position
- Apodex wants to solve hard problems directly rather than only sell tokens or build another chat assistant.
- Its near-term path is a post-training and harness loop: diagnose model weaknesses, construct tasks and training data, train, verify, and adjust the harness.
- Its verification strategy includes code tests, formal math verification where possible, and [[MultiAgentCollaboration]] when no single ground truth exists.
- Its first high-value domains in the source include biology, medicine, drug discovery, old-drug repurposing, and diagnosis.
- [[ChenTianqiao]] is described as shaping both strategic direction and model behavior through a constitution-like set of priorities.

## Connections
- [[ChenTianqiao]], [[DuShaolei]], and [[LiBeibin]] — founder/strategic leader and chief scientists in the source.
- [[RecursiveSelfImprovement]], [[DiscoveryModel]], [[DeepResearch]], and [[AIVerification]] — main technical themes.
- [[ResearchTaste]], [[ProblemDefinitionInResearch]], and [[AIForScience]] — scientific-discovery constraints.
