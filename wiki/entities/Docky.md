# Docky

Docky is [[ZhangYueguang]]'s current agent product direction in [[130-zhang-yueguang-chuangye-liangnian-shouci-fangtan-miaoya-bushi-ai-native-chanpin-liucheng-dao-shangxiawen-sheji-one-way-door-he-yinv-youxi]]. The first wedge is PPT generation, chosen because the category has product-market fit but existing AI PPT tools have not become [[OneWayDoorProduct]] experiences.

The source frames Docky as an [[AIFriendProducts]] attempt rather than a pure automation tool. Zhang wants an agent that helps users cross ability boundaries, with practical value and emotional attachment gradually blending when the same object repeatedly helps solve hard tasks. His product approach is to refine the smallest generated unit and needed context first, then reverse-engineer the user flow.

## Product Claims
- PPT is a useful entry because users already need presentations, but templates, code-generated slides, and image-generated slides each create serious editability or quality limits.
- The intended agent value is "ability breakthrough," not only end-to-end task outsourcing.
- The agent loop may be short, frequent, and low-latency rather than only long-running offline delegation.
- The product should preserve human participation by expanding user capability instead of simply removing the user from production.
