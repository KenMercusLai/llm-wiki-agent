# Claude Code

Claude Code is discussed in [[gaoshou-zenme-yong-ai-putongren-zenme-xue-ai-touziren-ruhe-tou-ai-duitan-kedaibiao-lizheng-ljqyo4tz0o2-pmsl-mjx6umsuzsc]] as one of the agentic coding tools that move users beyond conversational prompting. The guest says he previously used it more heavily before shifting more work to [[Codex]]. [[agent-yuannian-di-500-tian-shenme-zai-xiaoshi-shenme-zai-dansheng-weishenme-women-bugai-zai-touzi-gui-siwei-de-ruanjian-lhwdxfpke3bmamjk4e6knk-5sn-b]] treats Claude Code as a milestone in the 500-day agent timeline and as a benchmark for projects such as [[CodePilot]] and [[YouyouAgent]]. [[renlei-he-ai-agent-de-zuijia-peihe-fangshi-hai-mei-bei-faming-duitan-paperboy-ltgxurpseowqggfvgc32aurymt-o]] adds Claude Code as a successful agent reference that [[Paperboy]] still sees as project/session oriented rather than deeply personalized through [[OSLevelContext]].

[[tan-mi-claude-code-gao-dong-agent-harness-dui-tan-lai-xin-lu-lkluk3i7c4gzw4jvxee7odsfgis3]] makes Claude Code the central sample for [[AgentHarness]] analysis. [[LaiXinlu]] highlights its tool use, markdown-based memory, skill-like file loading, stop-hook memory updates, AutoDream-style consolidation, context compression, and cross-agent handoff as evidence for a design philosophy of giving the model more context, more action capacity, and less brittle external control.

[[dang-women-zai-taolun-harness-de-shihou-women-zai-taolun-shenme-shendu-duitan-minimax-hermes-agent-lvhm1cfno7mqmfv3g0aajmw4zdpd]] adds Claude Code as a comparison point for the broader agent wave. The guests describe scheduled tasks, IM access, phone remote control, and Memory-folder direction as OpenCloud-like, discuss real-name requirements through [[AgentIdentityAndAuthentication]], and read source-code leakage as evidence that top agent products are still exploring similar harness problems rather than relying on hidden magic.

[[ep108-vibe-coding-da-dizhen-cursor-dingjia-zhengyi-windsurf-shougou-fengbo-moxing-changshang-qin-erzi-men-you-jiang-ruhe-jinchang-lqn-icq1xqgk7xxxxzrpunj4fan]] adds Claude Code as a direct [[ModelProviderToolCompetition]] pressure on [[Cursor]]. The hosts say users compare Claude Code favorably on complex codebase understanding, while also noting subscription limits, cross-border payment friction, and the likelihood that CLI-first tools will eventually need stronger GUI support for diffs, selective acceptance, and history recovery.

[[vol-166-xianliao-cong-gemini-dao-ai-de-jiasu-yu-hundun-1-6650-1]] adds Claude Code to an everyday orchestration workflow with [[Superpowers]] and [[Codex]]. The hosts treat Claude's planning ability as strong and describe subagents, review loops, and long tasks as useful but attention- and token-intensive.
