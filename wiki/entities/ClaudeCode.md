# Claude Code

Claude Code is discussed in [[gaoshou-zenme-yong-ai-putongren-zenme-xue-ai-touziren-ruhe-tou-ai-duitan-kedaibiao-lizheng-ljqyo4tz0o2-pmsl-mjx6umsuzsc]] as one of the agentic coding tools that move users beyond conversational prompting. The guest says he previously used it more heavily before shifting more work to [[Codex]]. [[agent-yuannian-di-500-tian-shenme-zai-xiaoshi-shenme-zai-dansheng-weishenme-women-bugai-zai-touzi-gui-siwei-de-ruanjian-lhwdxfpke3bmamjk4e6knk-5sn-b]] treats Claude Code as a milestone in the 500-day agent timeline and as a benchmark for projects such as [[CodePilot]] and [[YouyouAgent]]. [[renlei-he-ai-agent-de-zuijia-peihe-fangshi-hai-mei-bei-faming-duitan-paperboy-ltgxurpseowqggfvgc32aurymt-o]] adds Claude Code as a successful agent reference that [[Paperboy]] still sees as project/session oriented rather than deeply personalized through [[OSLevelContext]].

[[tan-mi-claude-code-gao-dong-agent-harness-dui-tan-lai-xin-lu-lkluk3i7c4gzw4jvxee7odsfgis3]] makes Claude Code the central sample for [[AgentHarness]] analysis. [[LaiXinlu]] highlights its tool use, markdown-based memory, skill-like file loading, stop-hook memory updates, AutoDream-style consolidation, context compression, and cross-agent handoff as evidence for a design philosophy of giving the model more context, more action capacity, and less brittle external control.

[[dang-women-zai-taolun-harness-de-shihou-women-zai-taolun-shenme-shendu-duitan-minimax-hermes-agent-lvhm1cfno7mqmfv3g0aajmw4zdpd]] adds Claude Code as a comparison point for the broader agent wave. The guests describe scheduled tasks, IM access, phone remote control, and Memory-folder direction as OpenCloud-like, discuss real-name requirements through [[AgentIdentityAndAuthentication]], and read source-code leakage as evidence that top agent products are still exploring similar harness problems rather than relying on hidden magic.

[[ep108-vibe-coding-da-dizhen-cursor-dingjia-zhengyi-windsurf-shougou-fengbo-moxing-changshang-qin-erzi-men-you-jiang-ruhe-jinchang-lqn-icq1xqgk7xxxxzrpunj4fan]] adds Claude Code as a direct [[ModelProviderToolCompetition]] pressure on [[Cursor]]. The hosts say users compare Claude Code favorably on complex codebase understanding, while also noting subscription limits, cross-border payment friction, and the likelihood that CLI-first tools will eventually need stronger GUI support for diffs, selective acceptance, and history recovery.

[[vol-166-xianliao-cong-gemini-dao-ai-de-jiasu-yu-hundun-1-6650-1]] adds Claude Code to an everyday orchestration workflow with [[Superpowers]] and [[Codex]]. The hosts treat Claude's planning ability as strong and describe subagents, review loops, and long tasks as useful but attention- and token-intensive.

[[ep124-weishenme-agent-shidai-cli-faner-chengle-zuiyoujie-lufh0-oxxxqthj-guc7o-1mexuax]] mentions Claude Code as an environment that can install [[Podwise]] CLI/Skills, reinforcing the existing theme that Claude Code-style agents benefit when outside products expose clear [[AgentOptimizedCLI]] commands.

[[ep127-cong-skills-dao-zidonghua-gongzuoliu-lun-agent-ruhe-jieguan-zhenshi-shengchanli-lntwhoxpi433ptke-nhohb-5lbpz]] adds Claude Code as both an implementation tool and a review counterpart. The hosts describe handing generated landing-page designs or assets to Claude Code or [[Codex]], and also using cross-agent review where one agent writes code and the other critiques it.

[[1-yi-token-julebu-jibaole-ai-de-ranliao-bugoule-duitan-yu-wenyuan-aliyun-bailian-jishu-fuzeren-ltn5k9jd9e04i5mfdkdo-ycoslsm]] uses Cloud Code or Claude Code-style coding agents as a demand-side signal: coding agents move AI from testing into production work and can consume enough model capacity that [[MaaSInfrastructure]] must plan around peak usage, latency, and serving cost.

[[ba-ai-chuicheng-hewuqi-de-ren-qinshou-laxiale-xinlengzhan-tiemu-1]] adds an organizational note about Anthropic's internal Claude Code work. The hosts say the Claude Code client team is small, Anthropic uses AI heavily internally, and multiple internal projects compete; the source uses this as context for why workflow and organization can matter as much as headline model access.

[[vol-167-token-ru-liushui-agent-si-chaoyang-1-6653-1]] adds Claude Code as the main comparison point for [[Codex]] browser access and coding quality. The hosts say the gap between Codex and Claude Code has narrowed, discuss Claude Code extension/account-risk concerns, and tie heavy use of both tools to [[AIInferenceCostStructure]].

[[vol-162-keji-kuaile-xingqiu-44-xin-moxing-sotamen-qihe-xinchun-1-6628-1]] adds a sharper [[ModelWorkflowFit]] comparison. The hosts describe Claude Code as faster and more willing to infer the user's intent than [[Codex]], making it useful for clearer small tasks, while also implying that this tendency needs tighter review when the task is ambiguous or high-risk.

[[e155-sihu-meishenme-ren-zai-ti-ai-paomolun-le-lkon87vgpkdkq9ll-fg0eabnuubf]] adds Claude Code as part of Anthropic's enterprise-agent route. The episode treats coding as the first broadly successful agent domain because code has standardized inputs, fast feedback, and testable outputs; Claude Code and related work products become proof points for [[AgenticWorkflow]], [[AISkills]], and [[AIInvestmentMetrics]].

[[dang-kekaode-daima-biancheng-le-ou-er-fafeng-de-openclaw-women-weilai-de-gongzuo-fanshi-bianqian]] adds Claude Code through memory files, skills, and context-loss mitigation. The hosts discuss Claude Code-style long-term markdown memory for project rules, running logic, and test methods, then contrast that with cases where agents forget earlier fixes or reintroduce bugs unless the workflow is engineered around modules, memory, and verification.

[[140-dui-yao-shunyu-de-4-xiaoshi-fangtan-qing-yunxu-wo-xiao-feng-yixia-zai-anthropic-he-gemini-xun-moxing-jishu-yuce-yingxiongzhuyi-yi-guoqu-ll7qiciwwgfssorhr4yy-uuqae8h]] adds an ex-Anthropic researcher view. [[YaoShunyu]] says Anthropic's earlier business looked vulnerable when revenue was mainly API tokens, but Claude Code and adjacent work-efficiency products later made the product route more convincing. He also treats coding as the first major agent domain because it has unusually clear feedback and verification.

[[142-yusen-de-chuangtou-guancha-di-2-ji-harness-xia-yige-zijie-2026-da-jihui-he-stanley-druckenmiller-lg4sphlaunrjuulqraxs-1gc5ufz]] adds [[DaiYusen]]'s investor and product reading. He treats Claude Code as a turning point that showed agentic coding had crossed a practical threshold sooner than he expected, while also using it as proof that an [[AgentHarness]] can build user habit, configuration migration cost, memory, and workflow data beyond raw model access.

[[136-quanqiu-da-moxing-jibao-di-9-ji-he-guang-miliao-coding-shi-agi-di-er-mu-guigu-yusanjia-zhenxiang-moxing-zheng-chengwei-xin-yidai-os-lh-cqyoss-dztmyb5kmbjapa6w9v]] makes Claude Code the emblem of AGI's second act. The source says frontier researchers and strong programmers increasingly direct AI-written code rather than writing everything themselves, and it treats Claude Code as evidence that coding agents can accelerate AI research, model-company revenue, and [[ModelAsOperatingSystem]] platform formation.

[[263-sora-si-le-adobe-die-le-meitu-he-qu-he-cong-lgjmyveooc8wpzr0yviggvzvdyfs]] uses Claude Code as an analogy for application-layer pressure outside coding. The hosts treat [[ClaudeCode]], [[Codex]], and [[Cursor]] as the most visible case where model providers enter a high-value workflow directly, then extend that logic to [[Sora]], [[Adobe]], and [[Meitu]] in creative tools.
