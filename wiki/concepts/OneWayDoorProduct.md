# One Way Door Product

One way door product is [[ZhangYueguang]]'s standard for a product whose new solution is so much better that users do not want to return to the old method. In [[130-zhang-yueguang-chuangye-liangnian-shouci-fangtan-miaoya-bushi-ai-native-chanpin-liucheng-dao-shangxiawen-sheji-one-way-door-he-yinv-youxi]], he uses it to evaluate [[Miaoya]] and, more strongly, the still-unproven ambition for [[Docky]].

The concept matters because AI demos can look impressive without changing default behavior. A one-way-door AI product must not only generate something novel; it must create a new user expectation around speed, quality, editability, control, emotional payoff, or ability expansion that makes the previous workflow feel obsolete.

## Signals
- Users tolerate onboarding, pricing, or workflow change because the new result is visibly better.
- The product changes repeat behavior rather than producing one-time curiosity.
- The product's value is grounded in a real job or emotional loop, not only in AI novelty.
- For agent products, the system expands what the user can do while preserving enough review and control to be trusted.
