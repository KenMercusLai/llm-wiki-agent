# Preregistration

Preregistration is the practice of recording hypotheses, methods, and analysis plans before doing or completing a study. [[dont-hate-the-replicator-hate-the-game]] mentions it as one reform social scientists were already using to respond to the [[ReplicationCrisis]].

The source treats preregistration as useful but incomplete. It can reduce [[PHacking]] by fixing some decisions before researchers see outcomes, but it does not replace [[ReplicationPackage|replication packages]], [[RobustnessChecks]], or institutions that make follow-up inspection credible.

## Key Claims
- Preregistration separates planned tests from after-the-fact exploration.
- It can lower the temptation to tune analysis around a [[StatisticalSignificanceThreshold]].
- The practice is strongest when later readers can compare the preregistered plan with the actual analysis.
- It does not solve weak data documentation or coding errors by itself.
- [[ResearchIntegrityIncentives]] matter because a preregistration only constrains behavior if deviations can be noticed.
