# Context Flywheel

Context flywheel is [[Tristan]]'s product-growth loop for [[Elys]] in [[135-he-ziran-xuanze-chuangshiren-tristan-liao-elys-saibo-fenshen-linghun-context-de-huoqu-yu-liudong-he-ai-shejiao-wangluo-ltwegwvo7grn-v-rft0txlmqmcty]]. The loop is: user interactions create more context, the [[CyberAvatars]] becomes more like the user, the avatar brings back better matches or conversations, and visible value encourages the user to provide more context.

The idea extends [[ContextEngineering]] from a static prompt or memory file into a consumer-product loop. Context is not only stored; it is earned, refined, tested against outcomes, and made valuable enough that users accept the privacy and effort cost of supplying it.

## Key Claims
- Context flywheels must solve two cold starts at once: enough user-specific context and enough network density for matching.
- User actions such as browsing, commenting, approving avatar comments, and assigning tasks can become context if the product asks for confirmation and updates memory.
- Real-world outcomes, such as successful meetups or useful introductions, are stronger flywheel proof than in-app novelty alone.
- The loop can fail if onboarding is too long, avatar control feels weak, comments become homogeneous, or privacy cost exceeds visible benefit.
